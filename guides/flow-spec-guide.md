# FlowFrame 플로우 스펙 작성 가이드

FlowFrame에 업로드할 플로우 스펙 마크다운 작성 규칙입니다.

---

## 전체 구조

```markdown
# 서비스명

## Screens

### 그룹명
- slug: 화면명
- slug: 화면명 (modal)
- slug: 화면명 (external: https://example.com)

## Flow
출발slug → 도착slug: 사용자 행동 설명
```

파일은 크게 **Screens** 섹션과 **Flow** 섹션 두 부분으로 구성됩니다.

---

## Screens 섹션 규칙

### 기본 형식

```markdown
## Screens

### 그룹명
- slug: 화면명
```

| 요소 | 규칙 |
|------|------|
| `## Screens` | **필수**. 정확히 이 이름이어야 합니다. |
| `### 그룹명` | 화면들을 묶는 카테고리. 한글/영문 자유. |
| `slug` | **영문 소문자**로 시작, 소문자·숫자·하이픈만 허용. |
| `화면명` | 캔버스에 표시되는 이름. 한글 가능. |

### 화면 타입

| 타입 | 작성법 | 설명 |
|------|--------|------|
| 일반 화면 | `- slug: 화면명` | 기본. 별도 표기 없음. |
| 모달/팝업 | `- slug: 화면명 (modal)` | 괄호 안에 `modal` |
| 외부 링크 | `- slug: 화면명 (external: URL)` | 괄호 안에 `external: URL` |

### 주의사항

- slug는 **파일 전체에서 고유**해야 합니다. 중복 시 에러.
- `(popup)`, `(dropdown)`, `(tab)`, `(floating)` 등은 인식되지 않습니다. **`(modal)`만 지원**합니다.
  - 팝업, 드롭다운, 플로팅 등도 모달로 통일하거나, 타입 없이 일반 화면으로 등록하세요.
- 그룹명에 `>` 같은 계층 표현(`상품 > B tv`)은 사용 가능하지만, 캔버스에서는 단일 그룹으로 표시됩니다.

---

## Flow 섹션 규칙

### 기본 형식

```markdown
## Flow
출발slug → 도착slug: 사용자 행동 설명
```

| 요소 | 규칙 |
|------|------|
| `## Flow` | **필수**. 정확히 이 이름이어야 합니다. |
| `→` | **유니코드 화살표**(U+2192)만 허용. `->` 는 인식되지 않습니다. |
| `출발slug` / `도착slug` | Screens에 정의된 slug와 정확히 일치해야 합니다. |
| `액션 설명` | 콜론(`:`) 뒤에 작성. 사용자 행동을 간결하게. |

### 주의사항

- **Flow 섹션에 `###` 소제목을 쓰지 마세요.** 파서가 인식하지 못합니다.
  - `→` 가 포함된 소제목은 파싱 에러를 발생시킵니다.
  - 그룹핑이 필요하면 **빈 줄**로 구분하세요.
- **양방향 관계를 피하세요.** `A → B`와 `B → A`를 동시에 쓰면 캔버스에서 엣지가 겹칩니다.
  - 취소, 닫기, 뒤로가기 같은 역방향은 생략하세요.
  - 필수 역방향(예: 비밀번호 재설정 → 로그인)만 남기세요.
- Screens에 정의되지 않은 slug를 Flow에서 참조하면 에러입니다.

---

## 자주 하는 실수

| 실수 | 예시 | 해결 |
|------|------|------|
| Flow에 소제목 사용 | `### 홈 → 진입` | 소제목 삭제. 빈 줄로 구분. |
| 지원하지 않는 타입 | `(popup)`, `(tab)`, `(floating)` | `(modal)`로 통일하거나 타입 생략. |
| 화살표 오타 | `->`, `=>`, `-->` | `→` (유니코드) 사용. |
| slug 대문자 | `Login`, `MY-Dashboard` | `login`, `my-dashboard` |
| slug에 한글 | `- 로그인: 로그인` | `- login: 로그인` |
| 양방향 관계 | `A → B` + `B → A` | 주요 방향만 남기기. |

## 올바른 예시

```markdown
# B world

## Screens

### GNB·공통
- gnb: GNB
- login: 로그인
- signup: 회원가입

### 홈
- home: 홈

### 상품
- product-list: 상품 목록
- product-detail: 상품 상세
- consult-popup: 상담 신청 (modal)

### MY
- my-dashboard: MY 대시보드
- my-fee-bill: 요금안내서

## Flow
gnb → home: 로고 클릭
gnb → login: 로그인 클릭
gnb → my-dashboard: MY 클릭

home → product-list: 상품 메뉴 클릭
product-list → product-detail: 상품 카드 클릭
product-detail → consult-popup: 상담신청 클릭

login → my-dashboard: 로그인 성공
my-dashboard → my-fee-bill: 요금안내서 클릭
```

---

## 파일 업로드

1. 위 규칙에 맞게 `.md` 파일을 작성합니다.
2. FlowFrame 캔버스 상단 **스펙 업로드** 버튼을 클릭합니다.
3. 파싱 에러가 있으면 에러 메시지와 줄 번호가 표시됩니다.
4. 성공하면 화면 노드와 관계가 캔버스에 자동 배치됩니다.
