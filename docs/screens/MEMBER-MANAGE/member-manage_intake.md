## 화면 목적

서비스 운영팀 관리자가 회원 목록을 조회·검색·필터링하고, 회원 상세 정보를 확인·수정하며, 회원 상태 및 역할을 관리하는 화면

## 핵심 행동

- 회원 목록을 검색·필터링하여 조회 (이름, 이메일, 가입일, 상태)
- 회원 상세 정보를 보고 수정 (프로필, 메모)
- 회원 상태를 변경 (활성/정지/탈퇴)
- 회원에게 역할을 부여/해제 (관리자, 일반, VIP)

## 화면 구성

좌측 회원 목록 패널 + 우측 상세 패널 (패널 분할)

## 모달

- 상태 변경 확인 — 회원 상태를 변경할 때 사유 입력 (화면 종속)
- 역할 변경 — 역할 부여/해제 시 확인 (화면 종속)

## 특수 인터랙션

인라인 편집 (메모 필드)

## viewport

둘 다

## 제약사항

- 상태 변경 시 사유 필수 입력
- 본인 계정은 상태 변경 불가
