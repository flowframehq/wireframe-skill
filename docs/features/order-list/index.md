# 주문 목록

## 와이어프레임 요소

| 요소 | type | 설명 |
|------|------|------|
| 검색바 | input | 주문번호, 주문자명, 연락처로 검색 (placeholder: "주문번호, 주문자명, 연락처 검색") |
| 주문상태 필터 | select | 전체/결제대기/결제완료/배송준비/배송중/배송완료/취소/환불요청/환불완료 |
| 결제방법 필터 | select | 전체/신용카드/계좌이체/간편결제/가상계좌 |
| 기간 필터 | input | 시작일~종료일 날짜 범위 선택 (date range picker) |
| 주문 테이블 | table | 컬럼: 주문번호, 주문일시, 주문자명, 상품명(대표+외 N건), 결제금액, 결제방법, 주문상태, 배송상태 |
| 정렬 토글 | button | 각 컬럼 헤더 클릭 시 오름차순/내림차순 정렬 |
| 페이지네이션 | button | 페이지 번호 이동, 이전/다음 버튼 |
| 주문 건수 표시 | text | "총 N건" 검색/필터 결과 건수 |

## 상태

| 상태 | 설명 |
|------|------|
| 기본 | 전체 주문 목록이 최신순으로 표시 |
| 빈 상태 | "조건에 맞는 주문이 없습니다" 메시지 |
| 로딩 | 테이블 스켈레톤 UI (5행 분량) |
| 에러 | "주문 목록을 불러오지 못했습니다" + 재시도 버튼 |
| 검색/필터 적용 | 필터 조건 태그 표시, 결과 건수 갱신 |

## 인터랙션

- 검색바에 키워드 입력 후 Enter 또는 검색 아이콘 클릭 → 결과 필터링
- 필터(주문상태, 결제방법, 기간) 변경 → 즉시 테이블 갱신
- 컬럼 헤더 클릭 → 해당 컬럼 기준 정렬 토글 (오름차순 ↔ 내림차순)
- 테이블 행 클릭 → 주문 상세 드로어 열림
- 페이지네이션 클릭 → 해당 페이지 데이터 로드
- 필터 초기화 버튼 → 모든 필터/검색 조건 리셋

## 유저스토리

- 쇼핑몰 관리자로서 전체 주문 현황을 한눈에 파악하고 싶다, 빠르게 처리해야 할 주문을 찾기 위해
- 쇼핑몰 관리자로서 특정 주문을 검색하고 싶다, 고객 문의에 즉시 응대하기 위해
- 쇼핑몰 관리자로서 상태별로 주문을 필터링하고 싶다, 배송 준비가 필요한 주문만 모아보기 위해

## 인수조건

- Given 주문 관리 페이지에 접근하면 When 페이지가 로드되면 Then 전체 주문이 최신순으로 표시된다
- Given 검색바에 주문번호를 입력하고 When 검색하면 Then 해당 주문만 표시된다
- Given 주문상태 필터에서 "배송준비"를 선택하면 When 필터가 적용되면 Then 배송준비 상태 주문만 표시된다
- Given 기간 필터에서 날짜 범위를 설정하면 When 필터가 적용되면 Then 해당 기간 내 주문만 표시된다
- Given 주문이 없는 조건으로 검색하면 When 결과가 0건이면 Then 빈 상태 메시지가 표시된다
- Given 테이블 행을 클릭하면 When 주문 상세 드로어가 열리고 Then 해당 주문의 상세 정보가 표시된다

## 비즈니스 로직

- 기본 정렬: 주문일시 내림차순 (최신순)
- 페이지당 표시 건수: 20건
- 검색은 주문번호 완전일치, 주문자명/연락처 부분일치
- 필터 복수 적용 가능 (AND 조건)
- 주문상태 뱃지 색상: 결제대기(회색), 결제완료(파랑), 배송준비(주황), 배송중(초록), 배송완료(검정), 취소(빨강), 환불요청(보라), 환불완료(회색)
