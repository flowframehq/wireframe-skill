# 주문 상세

## 와이어프레임 요소

| 요소 | type | 설명 |
|------|------|------|
| 드로어 헤더 | text | 주문번호 + 주문상태 뱃지 + 닫기 버튼 |
| 주문 기본정보 | text | 주문번호, 주문일시, 현재 주문상태 |
| 주문자 정보 | text | 이름, 연락처, 이메일 |
| 상품 목록 테이블 | table | 컬럼: 상품명, 수량, 단가, 소계 |
| 주문 합계 | text | 상품 합계, 배송비, 할인, 최종 결제금액 |
| 결제 정보 | text | 결제방법, 결제금액, 결제일시 |
| 배송지 정보 | text | 수령인, 연락처, 주소, 배송메모 |
| 상태 변경 이력 | list | 시각, 변경 상태, 담당자, 사유 목록 |

## 상태

| 상태 | 설명 |
|------|------|
| 기본 | 주문 전체 정보가 섹션별로 표시됨 |
| 로딩 | 드로어 내 스켈레톤 UI |
| 에러 | "주문 정보를 불러오지 못했습니다" + 재시도 버튼 |

## 인터랙션

- 드로어 닫기 버튼 클릭 또는 외부 영역 클릭 → 드로어 닫힘
- 주문자 연락처 클릭 → 연락처 복사
- 배송지 주소 클릭 → 주소 복사
- 상태 변경 이력 스크롤 → 최신 이력이 상단

## 유저스토리

- 쇼핑몰 관리자로서 특정 주문의 전체 정보를 한 화면에서 확인하고 싶다, 고객 문의에 정확히 응대하기 위해
- 쇼핑몰 관리자로서 주문 상태 변경 이력을 확인하고 싶다, 처리 경과를 추적하기 위해

## 인수조건

- Given 주문 목록에서 행을 클릭하면 When 드로어가 열리면 Then 해당 주문의 기본정보, 주문자, 상품, 결제, 배송지 정보가 모두 표시된다
- Given 드로어가 열린 상태에서 When 닫기 버튼을 클릭하면 Then 드로어가 닫힌다
- Given 상품이 여러 개인 주문이면 When 드로어에 표시되면 Then 상품 목록이 테이블 형태로 모든 상품을 표시한다
- Given 상태 변경 이력이 있으면 When 드로어에 표시되면 Then 변경 시각, 상태, 담당자, 사유가 시간순으로 표시된다

## 비즈니스 로직

- 드로어는 화면 우측에서 슬라이드 인
- 상품 소계 = 단가 x 수량, 합계 자동 계산 표시
- 금액은 원화(₩) 표기, 천 단위 콤마
- 상태 변경 이력은 최신순 정렬
