# 주문 처리

## 공통 상태

| 상태 | 설명 |
|------|------|
| 처리 가능 | 해당 액션을 실행할 수 있는 주문 상태 |
| 처리 불가 | 주문 상태 또는 조건이 맞지 않아 액션 비활성화 |
| 처리 중 | 액션 실행 후 서버 응답 대기 |

## 공통 인터랙션

- 액션 실행 전 → 확인 다이얼로그로 의사 확인
- 처리 완료 → 주문 상세와 주문 목록의 상태 즉시 갱신
- 처리 실패 → 오류 메시지 표시, 이전 상태 유지

## 공통 비즈니스 로직

- 모든 주문 처리 액션은 현재 주문 상태에 따라 활성화/비활성화 결정
- 하나의 주문에 동시에 여러 액션을 실행할 수 없음
- 처리 이력은 주문 상세에 자동 기록
