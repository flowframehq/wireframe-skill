# 주문 처리

## 공통 상태

| 상태 | 설명 |
|------|------|
| 결제대기 | 주문 생성 후 결제 미완료 |
| 결제완료 | 결제 확인됨, 배송 준비 전 |
| 배송준비 | 상품 포장/출고 준비 중 |
| 배송중 | 택배사에 인계 완료, 배송 진행 중 |
| 배송완료 | 수령 확인 |
| 취소 | 관리자 또는 고객에 의해 주문 취소됨 |
| 환불요청 | 고객이 환불을 요청한 상태 |
| 환불완료 | 환불 처리 완료 |

## 공통 인터랙션

- 상태 변경 시 → 변경 사유 입력 필수
- 상태 변경 후 → 주문 목록 및 주문 상세에 즉시 반영
- 되돌릴 수 없는 상태 변경 (취소, 환불완료) → 확인 다이얼로그 표시

## 공통 비즈니스 로직

- 상태 전이 규칙:
  - 결제대기 → 결제완료, 취소
  - 결제완료 → 배송준비, 취소, 환불요청
  - 배송준비 → 배송중, 취소
  - 배송중 → 배송완료
  - 배송완료 → 환불요청
  - 환불요청 → 환불완료
- 허용되지 않은 전이는 UI에서 버튼 비활성화
- 모든 상태 변경은 변경 이력(시각, 담당자, 사유) 기록
