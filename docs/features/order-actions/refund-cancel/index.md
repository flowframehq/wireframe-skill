# 환불/취소

## 와이어프레임 요소

| 요소 | type | 설명 |
|------|------|------|
| 환불/취소 사유 선택 | select | 고객 변심, 상품 불량, 배송 지연, 기타 |
| 상세 사유 입력 | input | 사유 상세 텍스트 영역 (필수) |
| 환불 금액 표시 | text | 원래 결제금액, 환불 예정 금액 표시 |
| 환불 금액 조정 | input | 부분 환불 시 금액 직접 입력 (기본값: 전체 금액) |
| 처리 버튼 | button | "취소 처리" 또는 "환불 처리" 실행 |
| 확인 다이얼로그 | dialog | 최종 확인 (환불 금액, 사유 요약 표시) |
| 처리 이력 | list | 이전 취소/환불 처리 이력 (시각, 금액, 사유, 담당자) |

## 상태

| 상태 | 설명 |
|------|------|
| 취소 가능 | 결제완료~배송준비 상태, 취소 처리 폼 표시 |
| 환불 가능 | 환불요청 상태, 환불 처리 폼 표시 |
| 처리 불가 | 이미 취소/환불완료 상태, 처리 이력만 표시 |
| 처리 중 | 버튼 비활성화 + 스피너 |
| 완료 | 처리 완료 토스트 + 상태 갱신 |

## 인터랙션

- 사유 선택 + 상세 입력 후 처리 버튼 클릭 → 확인 다이얼로그 표시
- 확인 다이얼로그에서 "확인" → 취소/환불 실행
- 부분 환불 시 금액 직접 입력 → 원래 금액 이하만 허용
- 처리 완료 → 주문 상태 자동 전이 (취소 또는 환불완료)

## 유저스토리

- 쇼핑몰 관리자로서 고객의 취소 요청을 처리하고 싶다, 빠르게 취소 절차를 완료하기 위해
- 쇼핑몰 관리자로서 환불 금액을 조정하고 싶다, 부분 환불 등 상황에 맞게 처리하기 위해
- 쇼핑몰 관리자로서 취소/환불 이력을 확인하고 싶다, 분쟁 시 근거 자료로 활용하기 위해

## 인수조건

- Given 결제완료 상태 주문에서 When 취소 처리를 실행하면 Then 주문 상태가 "취소"로 변경된다
- Given 환불요청 상태 주문에서 When 환불 처리를 실행하면 Then 주문 상태가 "환불완료"로 변경된다
- Given 환불 금액을 원래 결제금액보다 크게 입력하면 When 처리 버튼을 클릭하면 Then 유효성 오류가 표시된다
- Given 사유를 입력하지 않고 When 처리 버튼을 클릭하면 Then 사유 필수 안내가 표시된다
- Given 이미 취소된 주문에서 When 환불/취소 영역을 보면 Then 처리 이력만 표시되고 폼은 비활성화된다

## 비즈니스 로직

- 취소는 결제완료~배송준비 상태에서만 가능
- 환불은 환불요청 상태에서만 처리 가능
- 부분 환불 허용 (환불 금액 ≤ 원래 결제금액)
- 사유 선택 + 상세 사유 모두 필수
- 취소/환불 처리 후 되돌릴 수 없음 (확인 다이얼로그 필수)
- 처리 이력에 담당자(관리자) 정보 자동 기록
