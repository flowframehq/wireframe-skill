# 주문 메모

## 와이어프레임 요소

| 요소 | type | 설명 |
|------|------|------|
| 메모 목록 | list | 작성자, 작성일시, 내용 표시 |
| 메모 입력 | input | 텍스트 입력 영역 (placeholder: "메모를 입력하세요") |
| 등록 버튼 | button | 메모 저장 |
| 삭제 버튼 | button | 본인 작성 메모에만 표시 |

## 상태

| 상태 | 설명 |
|------|------|
| 기본 | 메모 목록 최신순 표시, 입력창 비어있음 |
| 빈 상태 | "등록된 메모가 없습니다" 메시지 |
| 등록 중 | 등록 버튼 비활성화 + 스피너 |

## 인터랙션

- 메모 입력 후 등록 버튼 클릭 → 메모 목록 상단에 추가, 입력창 초기화
- 삭제 버튼 클릭 → "삭제하시겠습니까?" 확인 후 삭제
- 메모 내용이 비어있을 때 등록 버튼 클릭 → 입력 필수 안내

## 유저스토리

- 쇼핑몰 관리자로서 주문에 내부 메모를 남기고 싶다, 다른 관리자에게 처리 사항을 공유하기 위해
- 쇼핑몰 관리자로서 이전 메모를 확인하고 싶다, 주문 처리 맥락을 파악하기 위해

## 인수조건

- Given 메모를 입력하고 When 등록 버튼을 클릭하면 Then 메모 목록 상단에 새 메모가 추가된다
- Given 메모가 없는 주문에서 When 메모 영역을 보면 Then "등록된 메모가 없습니다" 메시지가 표시된다
- Given 본인이 작성한 메모에서 When 삭제를 클릭하고 확인하면 Then 해당 메모가 삭제된다
- Given 다른 관리자가 작성한 메모에서 When 메모를 보면 Then 삭제 버튼이 표시되지 않는다

## 비즈니스 로직

- 메모는 관리자 내부용 (고객에게 노출되지 않음)
- 본인 작성 메모만 삭제 가능
- 메모 최대 500자
- 최신순 정렬
